Resumen

Realizaba la integracion de paginas web para el procesamiento de pagos Online, dando soporte o colaborando con sus equipos para realizar correctamente la integracion.

Funciones

Programacion en Laravel

La API hacia de puente entre la pasarela y las paginas web, brindando una solucion completa para los clientes que debian realizar la transicion entre la antigua y nueva API de la pasarela.

Desarrollo y actualizacion plugin WordPress

Teniamos dos plugin, uno solo WordPress y otro con WooCommerce, estos podian comunicarse directamente con la pasarela, realizando la solicitud de los formularios y procesando los callbacks recibidos por esta.

Parte de mi trabajo fue tambien hacer la actualizacion de estos, ya que solo se comunicaban con la antigua API de la pasarela.

Desarrollo de funciones para panel administratibo

Contabamos con un panel de Sherpabots en el cual los clientes podian consultar el estado de sus paginas, por ejemplo las suscripciones activas actualmente con ellos.

La creacion de modulos y apps se realizaba con las herramientas brindadas por Sherpabots, pero la presentacion de la informacion se realizaba con MySQL.

Desarrollo y colaboracion

Realizaba desarrollos para la correcta solicitud de formularios de pago y procesamiento de callbacks, utilizando las tecnologias requeridas por los clientes, en su mayoria fue PHP.

En ocaciones tambien debia participar en el desarrollo de funcionalidades.

Por ejemplo la que mas tiempo trabaje, requerian ayuda para acelerar la puesta en marcha de la pagina, debia realizar ajustes varios de diseño, maquetando y añadiendo un tema oscuro a la pagina, tambien añadiendo nuevas funciones como un chat en vivo con el que debian comunicarse con el streamer. Este trabajo fue aprendiendo Next.Js y Sass, que eran los que utilizaban en la web.

Soporte tecnico

Haciendo uso del Ticketing de Sherpabots, me comunicaba con los clientes ayudandoles con los problemas que les podian surgir a ellos o su equipo.

Realizaba soporte a las incidencias sucedidas con la API nuestra, la API de la pasarela, el plugin y el panel administrativo de Sherpabots, tanto la pasarela como Sherpabots hacia de intermediario con sus equipos tecnicos.

Un poco mas

La pasarela principal con la que trabajabamos ofrecia SOAP antes para la solicitud de formularios de pago y callbacks, ahora ofrecen una solucion REST, añadiendo mas funcionalidades y permitiendo a los clientes personalizar mucho mas las solicitudes que se les realizan, por ejemplo brindan una configuracion muy completa para las suscripciones, permitiendo ser configuradas estas con la solicitud en JSON.

Cuando inicie en la empresa me encontraba aprendiendo mas de PHP y con unos conocimientos basicos de Laravel, debia empezar realizando la actualizacion de los plugin WordPress que ya tenian, para funcionar con la nueva API de la pasarela, asi tambien familiarizandome con el funcionamiento de las pasarelas de pago, dando el paso de SOAP a REST.

Asisti a un curso de unos cuantos dias para aprender el funcionamiento de Sherpabots, luego de este se nos dio acceso a una documentacion, con la que debia realizar las apps y modulos que serian utilizados por los clientes, entre estos por ejemplo, podian consultar solicitud de formularios, suscripciones, cancelaciones, reembolsos, seguimiento de el proceso de integracion con la pasarela, envio de facturas.
